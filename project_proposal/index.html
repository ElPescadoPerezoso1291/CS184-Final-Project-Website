Smoke Simulation

Geovanni Mojica, Kyle Wong, Michael Huang, Ethan Tam

Summary

We want to make a physics-based model for smoke or other gaseous substances and be able to animate fluid-like motions much like those seen in animation or games. Physical models are preferable to procedural or key frame techniques because it will easily generate the swirling-like motion of smoke and it can also interact with other objects and forces naturally. We will be using a method proposed by the “Stable Fluid” paper that claims to create an unconditionally stable model that will allow real-time interaction.

Problem Description

We see fluids in animations all the time; almost every game has water in it and recently Avatar: Way of the Water made headlines with their novel ways of animating water. Animators used to make fluid-like motions with procedural or key frame techniques; however, that way is inefficient and it does not accurately model real-life fluid-motion. Hand drawing the way fluids swirl and flow is a tedious process and it would be much easier to use the physical equations to model it. The problem with solving physical equations for fluids is that they normally use unstable schemes that can diverge for large time-steps, which will create a lag for the user since it will have to restart with a smaller time-step. We want to get physically-accurate fluid-like motion that can be rendered quickly and efficiently to allow for real-time interaction. We can do this by using a method described in the paper “Stable Fluid” that proposes an unconditionally stable model. While this model is not the most accurate, it still produces the complex flows we desire and works nicely for our interactive graphical application. Importantly, it is stable with large time-steps so we can use it with real-time interaction. This would give an animator precise control over the animation and allow for quicker, more accurate, and cost-efficient way to animate fluids.

Goals and Deliverables

We plan to create a program for simulating smoke given a specific temperature of the environment and a desired density for the smoke. We want to be able to create an interactive simulation where we can guide the generation of smoke with mouse inputs. Specifically, whenever we click, smoke comes out of the location the cursor is at.

We also want to add in the ability to change parameters on the spot like the temperature and density. To measure the quality and performance of our system we currently plan to just visually confirm that the program is doing what we expect it to do, e.g. smoke coming out of the correct location, colliding with already generated smoke correctly, going at a velocity that seems reasonable given the temperature, doing the same for the size of the smoke cloud, and seeing how smooth our simulation is (hardly any lag).

Our ultimate goal is to see if it's possible to create a simulation that estimates the behavior of an incompressible, homogeneous fluid under the Navier-Stokes equations efficiently and quickly, which tends to be the problem with physic based simulations. For this reason, we plan to deliver an interactive simulation of smoke that is able to render quickly and if all things go well we hope to expand the simulation to include different substances such as water.
